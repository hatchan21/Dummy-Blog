Editorial

Throughout 2009, we have been extremely busy migrating our forecasting technology toward cloud computing, and today this hard work has been rewarded by the Windows Azure Partner Award that Lokad is extremely proud to receive from Microsoft.

Cloud computing not only brings our forecasting technology to a whole new level, but it also enables us to design client apps that are easier to deploy and to maintain such as Salescast.

The cloud also offers the possibility of inexpensive ad-hoc integrations matching your very specific forecasting needs, not yet covered by the Lokad apps. In such situations, we suggest you to take some REST and see if it improves your forecasting accuracy. In our experience, it does.

Best regards,
Joannes Vermorel, Founder at Lokad.

Salescast, data integration made way easier

Data integration had always represented the No1 issue faced by our clients. Forecasting issues were simply dwarfed by the sheer amount of data import issues.

We have been trying to add more and more data adapters to our client apps, reaching +20 supported apps, and yet we were still failing at properly importing data as there are thousands of apps in the market.

Stepping back, we asked ourselves: why should our clients even bother about data integration? Letting our prospects and clients face such issues was not acceptable. Thus, we decided to address the problem from a radically different angle: just grant Lokad an access to your database, and Lokad will figure out the rest. The project codenamed Salescast was born.

In order to solve data integration issues, Salescast uses a combination of technology and process. Salecast attempts to auto-detect the app. If the data format is recognized, it proceeds directly with the forecasting process. If the database is not recognized, then a Lokad engineer gets appointed to design the data adapter (at no charge!). In any case, the time spent on setup from the client perspective is minimal.

Get your forecasts and business reports online today with Salescast.

Take some REST and rely on our forecasts

About 2 years ago, we were announcing the release of API v2. Since beginning of May, Forecasting API v3 is live. Both LSSC and L3C have been already upgraded to API v3. As a primary benefit, the 1 hour delay between data upload and forecast download is no more. If your dataset is small, processing is now much faster.

The primary focus of this release is simplicity. Indeed, API v2 had incrementally grown to +20 web methods with a certain level of redundancy. Then, over the last 2 years, thanks to our growing partner network, we discovered many minor yet annoying glitches in API v2 related to popular programming environments (Java, Python, C++, Apex, ...). Thus, we made sure that API v3 would avoid designs that prove to be troublesome in some environments.

Also, API v3 comes with both REST and SOAP endpoints. Indeed, REST has emerged as the approach to ensure maximal interoperability in modern web-oriented enterprise apps, and we are very committed in staying up to the industry standards.

Few facts about API v3:

  • 8 web methods (and only 4 of them actually required to for production usage) while API v2 had +20 methods. Less methods mean less time to spend deciphering our API spec.
  • The .NET Forecasting Client for API v3 has been reduced about 1k lines of code against +10k lines of code for the previous API v2 version.
